```python
class Markov_Switching(pt.nn.Module):

    使用转移概率矩阵控制模型之间的切换
    每个模型以一定概率保持或切换到其他模型
    
    参数说明：
    ----------
    n_models: int - 模型数量（如8个模型）
    switching_diag: float - 保持当前模型的概率（如0.8）
    switching_diag_1: float - 切换到相邻模型的概率（如0.15）
```

**关键方法**:
- `__init__`: 构建转移概率向量
- `init_state`: 初始化模型状态（均匀分布或确定性）
- `forward`: 执行模型切换（根据转移概率采样）
- `get_regime_probs`: 获取切换概率矩阵

**数学原理**:
```
转移概率向量：
tprobs[0] = switching_diag      # 保持当前模型
tprobs[1] = switching_diag_1    # 切换到相邻模型
tprobs[其他] = (1 - switching_diag - switching_diag_1) / (n_models - 2)
```

```python
class DIMMPF(SSM):
    """
    深度交互多模型粒子滤波（Deep Interacting Multiple Model Particle Filter）
    
    论文的核心算法！
    
    特点：
    1. 端到端学习切换动力学（使用 NN_Switching）
    2. 端到端学习状态转移（使用神经网络替代线性方程）
    3. 可微分粒子滤波，支持梯度反向传播

    - RLPF: 学习切换，状态转移保持线性（x_t = a*x_{t-1} + b）
    - DIMMPF: 同时学习切换和状态转移（x_t = NN(x_{t-1}) + noise）
    """
```

**重点理解**:
```python
# M_t_proposal: 状态转移（使用神经网络）
def M_t_proposal(self, k, x_t_1, t: int):
    # 从辅助变量中提取切换信息
    r = x_t_1[:, :, 1:]
    # 使用神经网络计算状态转移
    # dyn_models[k]: 模型k的动态网络
    locs = self.dyn_models[k](x_t_1[:, :, 0:1])
    # 添加噪声
    noise = self.x_dist.sample(locs.shape) * self.sd_d
    new_pos = locs + noise
    # 更新辅助变量（切换动力学）
    r = self.switching_dyn.R_t(r)
    return pt.cat((new_pos, r), dim=2)

# log_f_t: 观测似然（使用神经网络）
def log_f_t(self, k, x_t, t: int):
    # 提取状态
    pos = x_t[:, :, 0:1]
    # 使用神经网络计算观测预测
    # obs_models[k]: 模型k的观测网络
    locs = self.obs_models[k](pos)
    # 计算高斯似然
    return self.var_factor * ((self.y[t] - locs) ** 2) + self.pre_factor
```

#### 2. 单步推进 advance_one (第 246-413 行) ⭐⭐⭐
```python
def advance_one(self):
    """
    单步推进粒子滤波器
    
    执行 IMM-PF 的核心算法（算法 10.3 的变体），推进一个时间步
    
    算法流程：
    1. 获取模型概率（切换概率）
       regime_probs = model.get_regime_probs(x_t)
    
    2. 调整模型概率（加入粒子权重）
       adj_regime_probs = regime_probs + true_weights[:,:,None]
    
    3. 计算总模型概率
       tot_regime_probs = pt.logsumexp(adj_regime_probs, dim=1)
    
    4. 计算重采样权重
       regime_resampling_weights = adj_regime_probs - tot_regime_probs[:, None, :]
    
    5. 对每个模型进行重采样
       for k in range(self.Nk):
           xs[k], new_weights[k], indices[k] = self.resampler(...)
    
    6. 状态转移（提议分布）
       x_t[k] = M_t_proposal(k, xs[k], t)
    
    7. 更新权重（观测似然 + 切换概率）
       log_weights = log_f_t(k, x_t[k], t) + new_weights[k] + tot_regime_probs
    
    8. 归一化权重
       true_weights = normalise_log_quantity(log_weights)
    """
```

**不同 IMM 类型的区别**:

- `'OT'`: 最优传输重采样，使用最优传输进行重采样

```python
for epoch in range(epochs):

            )
            DPF_redefined(simulated_object, T, likelihood_loss.get_reporters())
            likelihood_loss()
            # 设置损失权重：[1.0, lam] 表示 MSE 权重为1，ELBO 权重为λ
            complete_loss.register_data(
                weights=pt.tensor([1.0, lam], device=DPF.device)
            )
        
        # 4. 执行粒子滤波（主模型）
        DPF(simulated_object, T, complete_loss.get_reporters())
        
        # 5. 反向传播
        complete_loss()
        complete_loss.backward()
        
        # 6. 梯度裁剪（防止梯度爆炸）
        pt.nn.utils.clip_grad_value_(DPF.parameters(), clip)
        
        # 7. 参数更新
        opt.step()
```

## 💡 常见问题

**Q: 如何理解可微分性？**
A: 关键是通过重参数化技巧，使得粒子滤波的每个步骤都可以计算梯度。重点关注：
- `log_f_t` 和 `M_t_proposal` 方法
- `scale_grad` 类的梯度裁剪
- 重采样器的软重采样实现

**Q: 如何调试代码？**
A: 使用 PyCharm 或 VSCode 的调试器：
1. 在关键位置设置断点（如 `advance_one` 方法）
2. 观察张量的形状和值
3. 使用 `print(f"{tensor.shape}")` 打印形状
4. 使用 `tensor.detach().numpy()` 转换为 numpy 查看数值

**Q: 如何加速实验？**
A: 
1. 使用 GPU (`--device cuda`)
2. 减少粒子数和 epoch 数进行快速测试
3. 使用较小的网络（`--layers 2 --hid_size 8`）
4. 减少运行次数（`--n_runs 1`）

**Q: ELBO 损失是什么？**
A: ELBO（Evidence Lower Bound）是变分推断中的证据下界：
- 作用：衡量模型对数据的拟合程度
- 公式：ELBO = E[log p(x, y | θ)] - KL(q(θ) || p(θ))
- 在代码中：通过 `Magnitude_Loss` 和 `Log_Likelihood_Factors` 计算
- 与 MSE 的关系：总损失 = MSE + λ × ELBO

**Q: 不同 IMM 类型有什么区别？**
A:
- `'normal'`: 标准 IMM 实现，适用于传统粒子滤波
- `'new'`: 论文提出的新实现，支持端到端训练，使用 detach() 分离梯度
- `'OT'`: 最优传输重采样，保持梯度的连续性
